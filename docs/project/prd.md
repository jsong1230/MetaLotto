# MetaLotto — PRD

## 1. 프로젝트 개요

- **목적**: 메타디움 블록체인 기반 투명한 복권 DApp으로 META 토큰의 실질적 활용처 제공 및 생태계 활성화
- **대상 사용자**:
  - META 홀더 (기존 커뮤니티)
  - 일반 크립토 유저 (신규 유입)
  - 운영/관리자
- **핵심 가치**:
  - 온체인 추첨으로 100% 투명성 보장
  - META 토큰 실사용 처 증대
  - 커뮤니티 펀드를 통한 자생적 생태계 투자 재원 확보
  - 낮은 진입 장벽 (기존 지갑 사용, 낮은 가스비)

## 2. 핵심 기능 요약

| ID | 기능명 | 설명 | 우선순위 |
|----|--------|------|----------|
| F-01 | 라운드 관리 | 6시간 단위 라운드 생성/종료/추첨 자동화 | P0 |
| F-02 | Blockhash 난수 생성 | 미래 블록 해시 기반 당첨자 선정 (v2 VRF 업그레이드 경로 확보) | P0 |
| F-03 | 티켓 구매 | META 토큰으로 티켓 구매 (1장/복수 구매 지원) | P0 |
| F-04 | 상금 분배 | 90%/5%/5% 비율 분배 (당첨자/커뮤니티/운영) | P0 |
| F-05 | 자동 상금 지급 | 당첨 시 별도 클레임 없이 자동 전송 | P0 |
| F-06 | 웹 프론트엔드 | 지갑 연결, 티켓 구매, 라운드 현황, 결과 확인 DApp UI | P0 |
| F-07 | 이벤트 로그 | 모든 주요 액션이 이벤트로 기록되어 블록 익스플로러에서 확인 가능 | P0 |
| F-08 | 비상 정지 | Owner에 의한 컨트랙트 일시 중단 (Pause/Unpause) | P0 |

## 3. 비기능 요구사항

- **보안**:
  - ReentrancyGuard 적용 (buyTickets, drawWinner, claimRefund)
  - Pull Pattern으로 송금 실패 대비 (pendingWithdrawals)
  - 256블록 초과 시 forceCloseDraw() 복구 메커니즘
  - OpenZeppelin v5.x 기반 검증된 라이브러리 사용
- **성능**:
  - 티켓 최대 100장 제한으로 가스 폭발 방지
  - 단일 컨트랙트 구조로 컨트랙트 간 호출 가스 절약
- **확장성**:
  - IRandomnessProvider 인터페이스로 v2 VRF 업그레이드 경로 확보
  - 설정값(티켓 가격, 라운드 기간 등) Owner 변경 가능

## 4. 기술 스택

- **Smart Contract**: Solidity ^0.8.24 + Foundry + OpenZeppelin v5.x
- **Frontend**: Next.js 15 (App Router) + TypeScript + Tailwind CSS + wagmi/viem
- **Chain**: Metadium Mainnet (Chain ID: 11, EVM Compatible)
- **Token**: META (Native Token — msg.value로 처리)

## 8. 제약사항

- **외부 의존성**: 메타디움 RPC 엔드포인트 필수 (https://api.metadium.com/prod)
- **난수 생성**: v1은 blockhash 방식 사용 (풀 규모 확대 시 VRF 업그레이드)
- **blockhash 유효 범위**: 최근 256블록만 조회 가능 — 256블록 이내 drawWinner() 호출 필수
- **법적 제약**: 케이맨 제도 법인 기준 규제 준수 필요

## 9. 라운드 흐름

```
[Open] — 티켓 판매 중 (6시간, 하루 4라운드)
    ↓
[Closing] — 판매 종료, 추첨 트리거 (미래 블록 지정)
    ↓
[Drawing] — 지정 블록 도달, blockhash로 당첨자 선정
    ↓
[Completed] — 상금 자동 분배, 결과 기록
    ↓
[Next Round] — 새 라운드 자동 시작
```

## 10. 상금 분배 구조

```
Total Pool (100%)
├── 당첨자 상금: 90%
├── 커뮤니티 펀드: 5% (멀티시그 지갑)
└── 운영비: 5% (운영 지갑)
```
