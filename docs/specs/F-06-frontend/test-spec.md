## 단위 테스트 (Vitest + Testing Library)

### 1. WalletConnectButton 컴포넌트

| 시나리오 | 입력 | 예상 결과 |
|----------|------|-----------|
| 미연결 상태 렌더링 | `isConnected: false` | "지갑 연결" 버튼 표시 |
| 연결 상태 렌더링 | `isConnected: true, address: "0x1234...5678", balance: "100 META"` | 주소(축약) + 잔액 표시 |
| 연결 버튼 클릭 | 클릭 이벤트 | `connect()` 함수 호출 |
| 연결 해제 버튼 클릭 | 클릭 이벤트 | `disconnect()` 함수 호출 |

### 2. RoundInfoSection 컴포넌트

| 시나리오 | 입력 | 예상 결과 |
|----------|------|-----------|
| Open 상태 렌더링 | `status: 0 (Open)` | 녹색 "진행 중" 배지 |
| Closing 상태 렌더링 | `status: 1 (Closing)` | 노랑 "추첨 대기" 배지 |
| Completed 상태 렌더링 | `status: 2 (Completed)` | 회색 "완료" 배지 |
| Cancelled 상태 렌더링 | `status: 3 (Cancelled)` | 빨강 "취소" 배지 |
| 풀 규모 포맷팅 | `totalPool: 1000000000000000000000n` | "1000 META" 표시 |
| 티켓 수 포맷팅 | `ticketCount: 42n` | "42장" 표시 |
| 로딩 상태 | `isLoading: true` | 스켈레톤 UI 표시 |
| 에러 상태 | `error: ContractFunctionExecutionError` | 에러 메시지 표시 + 재시도 버튼 |

### 3. CountdownTimer 컴포넌트

| 시나리오 | 입력 | 예상 결과 |
|----------|------|-----------|
| 남은 시간 1시간 | `remaining: 3600n` | "01:00:00" 표시 |
| 남은 시간 0초 | `remaining: 0n` | "00:00:00" 표시 + "마감" 텍스트 |
| 실시간 카운트다운 | `remaining: 5n` | 1초마다 감소하여 "00:00:00" 도달 |

### 4. TicketPurchaseSection 컴포넌트

| 시나리오 | 입력 | 예상 결과 |
|----------|------|-----------|
| 기본 수량 | 초기 상태 | 수량 = 1 |
| 수량 증가 | "+" 버튼 클릭 | 수량 = 2 |
| 수량 감소 | "-" 버튼 클릭 (수량 2) | 수량 = 1 |
| 최소 수량 제한 | "-" 버튼 클릭 (수량 1) | 수량 = 1 유지 |
| 최대 수량 제한 | 수량 100에서 "+" 클릭 | 수량 = 100 유지 + "최대 100장까지 구매 가능합니다" 메시지 |
| 총 가격 계산 | `quantity: 5, ticketPrice: 100000000000000000000n` | "총 500 META" 표시 |
| 구매 버튼 비활성화 (미연결) | `isConnected: false` | 버튼 비활성화 + "지갑 연결 필요" 툴팁 |
| 구매 버튼 비활성화 (Closing) | `status: 1 (Closing)` | 버튼 비활성화 + "라운드 종료" 텍스트 |
| 구매 성공 | 구매 버튼 클릭 → 트랜잭션 성공 | 성공 토스트 + 수량 1로 리셋 |
| 구매 실패 (잔액 부족) | 구매 버튼 클릭 → `InsufficientPayment` 에러 | "잔액이 부족합니다" 에러 메시지 |

## 경계 조건 / 에러 케이스

### 1. 티켓 구매

- **수량 0**: 구매 버튼 비활성화, "최소 1장 이상 구매해 주세요" 메시지
- **수량 101**: "최대 100장까지 구매 가능합니다" 메시지
- **잔액 0 META**: 구매 버튼 비활성화, "META 잔액을 확인해 주세요" 메시지
- **잔액 부족 (99 META로 100장 구매 시도)**: "잔액이 부족합니다. 필요: 10000 META, 보유: 99 META" 메시지
- **라운드 Closed 상태**: 구매 버튼 비활성화, "현재 라운드가 종료되었습니다" 메시지
- **Pause 상태**: 구매 버튼 비활성화, "일시 중단되었습니다" 메시지

### 2. 지갑 연결

- **MetaMask 미설치**: "MetaMask를 설치해 주세요" 메시지 + 설치 링크
- **연결 거부**: "지갑 연결이 취소되었습니다" 메시지
- **잘못된 네트워크**: "Metadium 네트워크로 전환해 주세요" + 전환 버튼
- **연결 끊김**: "지갑 연결이 끊어졌습니다. 다시 연결해 주세요" 메시지

### 3. RPC 통신

- **RPC 타임아웃**: "네트워크 연결을 확인해 주세요" + 재시도 버튼
- **RPC 에러 (5xx)**: "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해 주세요" 메시지
- **잘못된 컨트랙트 주소**: "컨트랙트를 찾을 수 없습니다" 메시지

### 4. 환불/미수령 상금

- **환불 가능 티켓 없음**: `claimRefund` 호출 시 "환불할 티켓이 없습니다" 메시지
- **미수령 상금 없음**: `withdrawPending` 호출 시 "인출할 상금이 없습니다" 메시지
- **이미 환불 완료**: "이미 환불 처리되었습니다" 메시지

### 5. 이벤트 구독

- **WebSocket 연결 실패**: 폴링으로 자동 전환 (30초 간격)
- **이벤트 누락**: 수동 새로고침 버튼 제공

## E2E 테스트 (Playwright)

### 시나리오 1: 첫 방문 → 라운드 정보 확인

**목적**: 인수조건 1 - 페이지 로드 시 현재 라운드 정보 표시

```
1. 홈 페이지(/) 접속
2. 페이지 로딩 완료 대기
3. 라운드 상태 배지 확인 (녹색 "진행 중")
4. 카운트다운 타이머 표시 확인
5. 풀 규모 표시 확인 (예: "1000 META")
6. 참여 티켓 수 표시 확인 (예: "42장")
```

**검증 포인트**:
- [ ] 라운드 상태가 올바르게 표시됨
- [ ] 카운트다운이 실시간으로 감소함
- [ ] 풀 규모가 META 단위로 포맷팅됨
- [ ] 티켓 수가 "N장" 형식으로 표시됨

### 시나리오 2: 지갑 연결 → 잔액 확인

**목적**: 인수조건 2 - MetaMask 연결 시 지갑 주소와 META 잔액 표시

```
1. 홈 페이지 접속
2. "지갑 연결" 버튼 클릭
3. MetaMask 팝업에서 연결 승인
4. 지갑 주소 표시 확인 (0x1234...5678 형식)
5. META 잔액 표시 확인 (예: "1000 META")
6. 구매 버튼 활성화 확인
```

**검증 포인트**:
- [ ] 지갑 연결 버튼이 연결 상태로 변경됨
- [ ] 주소가 마스킹되어 표시됨 (0x1234...5678)
- [ ] 잔액이 META 단위로 표시됨
- [ ] 구매 버튼이 활성화됨

### 시나리오 3: 티켓 구매 성공

**목적**: 인수조건 3 - 티켓 수량 선택 및 구매 완료

```
1. 지갑 연결 상태로 홈 페이지 접속
2. 수량 "+" 버튼 클릭하여 5장 선택
3. 총 가격 확인 ("500 META")
4. "티켓 구매" 버튼 클릭
5. MetaMask 트랜잭션 승인
6. 트랜잭션 컨펌 대기
7. 성공 토스트 확인 ("구매가 완료되었습니다!")
8. 풀 규모 업데이트 확인
9. 티켓 수 업데이트 확인
10. 수량이 1로 리셋 확인
```

**검증 포인트**:
- [ ] MetaMask 트랜잭션이 올바른 값으로 생성됨
- [ ] 성공 메시지가 표시됨
- [ ] 풀 규모가 구매 금액만큼 증가함
- [ ] 티켓 수가 구매 수량만큼 증가함
- [ ] 수량이 1로 초기화됨

### 시나리오 4: 티켓 구매 실패 (잔액 부족)

**목적**: 에러 케이스 - 잔액 부족 시 에러 처리

```
1. 잔액 50 META인 지갑으로 연결
2. 수량 100장 선택 (필요: 10000 META)
3. "티켓 구매" 버튼 클릭
4. MetaMask에서 가스 추정 실패 또는 revert
5. 에러 메시지 확인 ("잔액이 부족합니다")
```

**검증 포인트**:
- [ ] 에러 메시지가 올바르게 표시됨
- [ ] 트랜잭션이 전송되지 않음
- [ ] UI 상태가 변경되지 않음

### 시나리오 5: 트랜잭션 취소

**목적**: 에러 케이스 - 사용자가 트랜잭션 거부

```
1. 지갑 연결 상태로 홈 페이지 접속
2. 수량 5장 선택
3. "티켓 구매" 버튼 클릭
4. MetaMask 팝업에서 "거부" 클릭
5. 에러 메시지 확인 ("트랜잭션이 취소되었습니다")
```

**검증 포인트**:
- [ ] 에러 메시지가 표시됨
- [ ] 수량이 유지됨 (리셋되지 않음)

### 시나리오 6: 실시간 참여 현황 업데이트

**목적**: 인수조건 4 - 실시간 참여 현황 표시

```
1. 홈 페이지 접속 (다른 브라우저/디바이스)
2. 별도 창에서 티켓 구매 실행
3. 원래 페이지에서 풀 규모 업데이트 확인
4. 티켓 수 업데이트 확인
```

**검증 포인트**:
- [ ] 다른 사용자의 구매가 반영됨
- [ ] 이벤트 구독이 정상 작동함
- [ ] UI가 자동으로 갱신됨

### 시나리오 7: 라운드 완료 → 당첨자 발표

**목적**: 인수조건 5 - 당첨자 정보 표시

```
1. Closing 상태의 라운드 페이지 접속
2. drawWinner() 실행 대기 (또는 이미 Completed 상태)
3. Completed 상태 전환 확인
4. 당첨자 주소(마스킹) 표시 확인
5. 당첨 금액 표시 확인
6. 새 라운드 시작 확인
```

**검증 포인트**:
- [ ] 상태가 Completed로 변경됨
- [ ] 당첨자 주소가 마스킹되어 표시됨 (0x1234...5678)
- [ ] 당첨 금액이 META 단위로 표시됨
- [ ] 새 라운드가 자동으로 시작됨

### 시나리오 8: 히스토리 페이지 조회

**목적**: 인수조건 6 - 과거 라운드 결과 표시

```
1. 히스토리 페이지(/history) 접속
2. 과거 라운드 목록 로딩 확인
3. 각 라운드 카드 정보 확인:
   - 라운드 ID
   - 당첨자 주소 (마스킹)
   - 당첨 금액
   - 일시
4. "더 보기" 버튼 클릭
5. 추가 라운드 로딩 확인
```

**검증 포인트**:
- [ ] 과거 라운드가 역순으로 표시됨
- [ ] 당첨자 주소가 마스킹됨
- [ ] 페이지네이션이 정상 작동함

### 시나리오 9: 내 티켓 페이지

**목적**: 사용자 참여 내역 및 환불/인출 기능

```
1. 내 티켓 페이지(/my-tickets) 접속 (미연결 상태)
2. "지갑 연결 필요" 메시지 확인
3. 지갑 연결
4. 현재 라운드 티켓 수 확인
5. 과거 라운드 참여 내역 확인
6. 미수령 상금이 있는 경우 "인출" 버튼 표시
7. 인출 버튼 클릭 → MetaMask 승인 → 완료
```

**검증 포인트**:
- [ ] 미연결 시 안내 메시지 표시
- [ ] 현재 라운드 티켓 수가 올바름
- [ ] 미수령 상금이 표시됨
- [ ] 인출이 정상적으로 처리됨

### 시나리오 10: 네트워크 에러 복구

**목적**: 에러 화면에서 재시도 버튼 동작

```
1. 홈 페이지 접속
2. 네트워크 차단 (오프라인 모드)
3. 에러 메시지 확인
4. "재시도" 버튼 클릭
5. 네트워크 복구
6. 데이터 다시 로딩 확인
```

**검증 포인트**:
- [ ] 에러 상태에서 재시도 버튼이 표시됨
- [ ] 재시도 버튼 클릭 시 데이터를 다시 조회함
- [ ] 복구 후 정상적으로 데이터가 표시됨

### 시나리오 11: 지갑 연결 해제 후 재연결

**목적**: 상태 갱신 시나리오

```
1. 지갑 연결 상태
2. MetaMask에서 연결 해제
3. "지갑 연결" 버튼으로 변경 확인
4. 구매 버튼 비활성화 확인
5. 다시 "지갑 연결" 버튼 클릭
6. MetaMask 승인
7. 이전 상태 복원 확인
```

**검증 포인트**:
- [ ] 연결 해제 시 UI가 즉시 반영됨
- [ ] 재연결 시 이전 데이터가 복원됨

### 시나리오 12: 취소된 라운드 환불

**목적**: 취소된 라운드에서 환불 처리

```
1. 취소된 라운드의 티켓 보유 상태
2. 홈 페이지 접속
3. "라운드가 취소되었습니다" 메시지 확인
4. "환불 받기" 버튼 표시 확인
5. 버튼 클릭 → MetaMask 승인 → 완료
6. META 잔액 증가 확인
```

**검증 포인트**:
- [ ] 취소 상태가 올바르게 표시됨
- [ ] 환불 버튼이 표시됨
- [ ] 환불이 정상적으로 처리됨
